# Astar Island Competition Code

This repository contains the source code used for the team's Astar Island submissions:

- live inference client and submission flow
- offline model training and benchmark tooling
- adaptive query selection logic for the last `5` queries
- tests and experiment history used to guide live deployment decisions

## Repository layout

- `astar_island/`: API client, inference pipeline, GBDT training code, priors, tiling logic
- `scripts/`: deadline automation, candidate-vs-stable selection, round submission helpers
- `tests/`: unit tests for terrain logic, prediction, tiling, learned priors, and GBDT helpers
- `models/`: current live bundle and selected reports kept with the codebase
- `HISTORY.md`: verified live results and deployment decisions
- `GAME_RULES.md`: local copy of competition rules used during development
- `DISCUS.md`: working notes and analysis

## Live policy implemented in code

- fixed live viewport: `15x15`
- full-map cover first: `45` queries for `5` seeds on a `40x40` map
- last `5` queries reserved for adaptive refinement
- predictions are always submitted for every seed in the active round
- if a serialized GBDT bundle is incompatible at runtime, the code falls back to the frequency prior instead of aborting the round

## Setup

Use Python `3.11+`.

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -e .
cp .env.example .env.local
```

Fill `.env.local` with your token before running the client.

## Environment variables

- `ASTAR_TOKEN`: bearer token from the platform
- `ASTAR_AUTH_MODE`: `bearer` or `cookie`
- `ASTAR_BASE`: API base URL, default `https://api.ainm.no`
- `ASTAR_CACHE_DB`: local sqlite cache path, default `.astar_cache.sqlite3`
- `ASTAR_TIMEOUT`: request timeout in seconds
- `ASTAR_INSECURE_SSL`: set to `1` only if you explicitly need insecure SSL locally

## Live commands

Inspect the active round and budget:

```bash
python3 -m astar_island.cli round-info
python3 -m astar_island.cli budget
python3 -m astar_island.cli status
```

Run the live pipeline:

```bash
python3 -m astar_island.cli run
```

Useful variants:

```bash
python3 -m astar_island.cli run --no-submit
python3 -m astar_island.cli run --force
python3 -m astar_island.cli run --output-dir runs/manual-test
python3 -m astar_island.cli run --skip-budget-check
```

## Offline commands

Train the main GBDT prior bundle:

```bash
python3 -m astar_island.cli train-model
```

Benchmark candidate vs stable bundles on the same recent history:

```bash
python3 -m astar_island.cli benchmark-models \
  --stable-model-path models/gbdt_prior.joblib \
  --candidate-model-path models/gbdt_v3_retrain2.joblib
```

Run backend bake-offs:

```bash
python3 -m astar_island.cli bakeoff-models --output-dir models/bakeoff
```

Run tests:

```bash
python3 -m unittest discover -s tests
```

## Notes on omitted local artifacts

The repository intentionally does not commit:

- `.env.local` and other local secrets
- `.venv/`
- `runs/` live payload dumps
- local sqlite cache files
- transient logs and bulky bake-off scratch outputs

Those artifacts are generated by the included source code and were kept local to avoid publishing secrets and unnecessary generated data.
